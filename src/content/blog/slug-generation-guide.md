## 什么是 Slug？为什么它对 SEO 至关重要？

在 URL `https://example.com/blog/how-to-use-slug-generator` 中，`how-to-use-slug-generator` 就是 slug。它是 URL 中标识特定页面的可读部分，直接影响用户体验和搜索引擎优化。

> 配套工具：[URL Slug 生成器](/slug)

### Slug 的三大价值

1. **用户可读**：比 `?id=12345` 更易理解，提升搜索结果点击率
2. **SEO 友好**：包含关键词的 slug 是 Google 排序的信号之一
3. **易于分享**：规范化的 URL 在社交媒体分享时更美观

## 一、Slug 设计的核心原则

### 1.1 只使用安全字符

slug 应仅包含：

- 小写字母 `a-z`
- 数字 `0-9`
- 分隔符（`-` 或 `_` 或 `.`）

所有空格、标点、特殊字符都应替换为分隔符或移除。

### 1.2 分隔符的选择

| 分隔符 | 示例 | SEO 评价 | 推荐场景 |
|--------|------|----------|----------|
| 连字符 `-` | `hello-world` | Google 推荐，视为单词分隔符 | 博客、通用网站 |
| 下划线 `_` | `hello_world` | Google 视为单词连接符，SEO 略差 | Django、特定框架 |
| 句点 `.` | `hello.world` | 较少见 | 文件名、命名约定 |

**Google 官方建议**：使用连字符（`-`）作为单词分隔符，Google 会将其识别为独立的单词。

### 1.3 长度控制

过长的 URL 会被搜索引擎截断，在搜索结果中显示不全。建议：

- slug 长度控制在 **60-80 字符**以内
- 在单词边界截断，避免截断到单词中间
- 截断后去除末尾残留的分隔符

## 二、中文 Slug 的处理策略

### 2.1 保留中文字符

现代浏览器和搜索引擎都能正确处理中文 URL。例如：

```
https://example.com/blog/如何使用slug生成器
```

浏览器会自动将中文部分编码为 `%XX` 形式传输，但地址栏仍显示原始中文。

**优点**：语义清晰，对中文用户友好
**缺点**：部分老旧系统或分享平台可能编码异常

### 2.2 移除非 ASCII 字符

另一种策略是移除所有非 ASCII 字符，仅保留英文部分：

```
输入：Hello World! 这是一个示例
输出：hello-world
```

适用于面向国际用户的站点，或纯英文内容场景。

### 2.3 拼音转换（需额外依赖）

部分站点会将中文转为拼音生成 slug，如 "这是一个标题" → `zhe-shi-yi-ge-biao-ti`。这需要引入拼音库，增加了项目复杂度。对于轻量化工具站，保留中文或移除非 ASCII 是更务实的选择。

## 三、停用词过滤

### 3.1 什么是停用词？

停用词（stopwords）是语言中高频但语义弱的词。英文常见停用词包括：

```
a, an, the, and, or, but, of, to, in, on, at, for, with, by, from, as, is, it, be, are, was
```

### 3.2 为什么要移除停用词？

移除停用词可以让 slug 更简洁、更聚焦核心关键词：

```
原始：how-to-use-the-slug-generator
过滤：use-slug-generator
```

### 3.3 停用词过滤的注意事项

- **仅对英文生效**：中文无停用词概念
- **可能改变语义**：极端情况下移除停用词可能导致歧义
- **不适用于短标题**：标题本身就很短时，移除停用词可能使 slug 过于简略

## 四、Slug 生成的算法流程

一个完整的 slug 生成器通常包含以下步骤：

```
1. 小写化（可选）→ 将所有字母转为小写
2. 分词 → 按非安全字符（空格、标点）分割为单词数组
3. 停用词过滤（可选）→ 移除停用词
4. CJK 处理 → 保留或移除中日韩字符
5. 连接 → 用分隔符连接单词
6. 截断（可选）→ 在最大长度处截断并清理末尾分隔符
```

### 4.1 Unicode 安全的分词

现代 slug 生成器使用 Unicode 属性转义来识别"安全字符"：

```javascript
// \p{L} 匹配所有语言的字母，\p{N} 匹配所有数字
const parts = text.split(/[^\p{L}\p{N}]+/u).filter(Boolean);
```

这种方式能正确处理中文、日文、韩文等非 ASCII 字符，比传统的 `/[a-z0-9]+/i` 更通用。

## 五、不同场景的 Slug 最佳实践

### 5.1 博客文章

```
标题：How to Build a RESTful API with Node.js
slug：how-to-build-restful-api-nodejs（移除停用词后更简洁）
```

### 5.2 电商商品页

```
商品名：Apple iPhone 15 Pro Max 256GB 深空黑
slug：apple-iphone-15-pro-max-256gb（移除颜色等变体属性，保留核心商品信息）
```

### 5.3 API 路由

```
资源：用户订单列表
slug：users-orders（RESTful 命名，复数形式）
```

### 5.4 文件名

```
标题：2026 年度财务报告（终稿）
slug：2026-financial-report-final（移除中文，转为安全文件名）
```

## 六、Slug 生成器的差异化设计

市面上多数 slug 生成器只提供基础的小写化 + 连字符替换。一个优质的 slug 工具应具备：

1. **多分隔符支持**：连字符、下划线、句点
2. **中文处理策略**：保留或移除可选
3. **停用词过滤**：可选移除英文停用词
4. **长度限制**：可滑动调节，在单词边界截断
5. **实时预览**：输入即生成，所见即所得
6. **全本地处理**：零上传零追踪

## 总结

URL slug 是 SEO 基础设施的重要一环。一个好的 slug 应该：简短、可读、包含关键词、使用连字符分隔、控制长度。对于中文站点，保留中文字符是可行的策略，但需注意部分平台的编码兼容性。通过 [URL Slug 生成器](/slug)，你可以快速将任意标题转为规范的 slug，提升站点的 SEO 表现与用户体验。
