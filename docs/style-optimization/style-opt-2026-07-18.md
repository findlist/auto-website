## 具体修改内容

### 1. `src/styles/global.css` - prose 行内代码边框统一

**问题**：原 `.prose code` 仅使用 `background: var(--color-bg-code)` + padding 标识行内代码，没有边框。在浅色背景下与周围正文区分度有限，且与 `.blog-article__content code`（已具备边框）的视觉风格不一致。`.prose pre code` 也未显式重置边框，存在被外层 pre 边框"双重包裹"的风险。

**优化**：

```css
.prose code {
  /* 其余属性保持不变 */
  border: 1px solid var(--color-border);
  padding: 1px 6px; /* 原 2px 6px，配合边框视觉收紧 */
}

.prose pre code {
  /* 其余属性保持不变 */
  border: 0; /* 避免与外层 pre 边框冲突 */
}
```

同步调整 `.json-faq details p code` 边框与 padding，与 prose 行内代码风格保持一致。

### 2. `src/styles/global.css` - FAQ summary 显式焦点环

**问题**：`.json-faq summary` 在键盘 Tab 导航时依赖浏览器默认 outline，与全站统一的 `var(--focus-ring)` box-shadow 焦点反馈风格不一致，且 summary 元素的 list-marker 区域会干扰默认 outline 的视觉表现。

**优化**：

```css
/* FAQ summary 显式焦点环：与表单元素/按钮统一的 box-shadow 焦点反馈 */
.json-faq summary:focus-visible {
  outline: none;
  box-shadow: inset 0 0 0 2px var(--color-primary);
  border-radius: var(--radius-md);
}
```

使用 `inset` 阴影贴合 summary 的折叠形态，避免外阴影被遮挡。

### 3. `src/styles/global.css` - 范围滑块统一样式

**问题**：密码工具的 `.pwtool__range` 与二维码工具的 `.qrtool__range` 使用原生 `<input type="range">`，在不同浏览器下视觉表现差异显著：WebKit/Blink 默认手柄偏小、Firefox 默认 track 颜色与全站设计令牌脱节。两处滑块风格不统一，且未对齐主色调。

**优化**：新增范围滑块统一样式块，同时覆盖 WebKit/Blink（`::-webkit-slider-runnable-track` / `::-webkit-slider-thumb`）与 Firefox（`::-moz-range-track` / `::-moz-range-thumb`）：

- 容器高度 24px
- track 使用 `var(--color-bg-code)` + `var(--color-border)` 边框
- thumb 为 18px 圆形，主色背景 + 2px 容器色边框
- hover 时 thumb 放大至 1.15，强化可供性
- focus-visible 时使用 `var(--focus-ring)` 统一焦点反馈
- 兼容 `prefers-reduced-motion`，禁用 transform 过渡

### 4. `src/styles/global.css` - 代码区选区颜色统一

**问题**：全站 `::selection` 已统一使用 `var(--color-primary-ring)` 作为选区背景，但代码块（`<pre>`）、JSON textarea、Diff textarea、正则输入框、Hash 输出区等场景下，由于背景色较深，选区颜色与背景对比度不足，且未体现品牌色调。

**优化**：为代码相关场景显式指定选区颜色，保证暗色背景下选区可见性与品牌一致性：

```css
.prose pre ::selection,
.blog-article__content pre ::selection,
.jsontool__textarea::selection,
.difftool__textarea::selection,
.regextool__pattern-input::selection,
.hashtool__output ::selection {
  background: var(--color-primary-ring);
  color: inherit;
}
```

### 5. `src/styles/global.css` - 打印样式（@media print）

**问题**：项目缺乏完整的打印样式。用户在浏览博客文章、工具结果时可能需要打印归档，但默认打印效果会包含 sticky header、footer、CTA 按钮、FAQ 折叠面板等交互元素，浪费纸张且影响可读性；同时链接 URL 丢失、代码块颜色失真。

**优化**：新增 `@media print` 样式块（约 95 行），覆盖以下场景：

**5.1 强制白底黑字**：去除暗色模式影响，确保打印可读性

**5.2 隐藏交互元素**：
- `.site-header` / `.site-footer` / `.skip-link` - 导航元素
- `.json-faq` - FAQ 折叠面板
- `.blog-article__cta` - 文末 CTA
- `.post-nav` - 上一篇/下一篇导航
- `.pagination` - 博客列表分页
- `.blog-tags-entry` - 标签入口
- `.hero__cta` - 首页 hero CTA
- `.tools__toolbar` - 工具栏
- `.btn` - 所有按钮

**5.3 容器与布局简化**：
- `.site-header` 取消 sticky 定位
- `.container` 去除 max-width 与 padding，让内容占满打印区域
- hero 区简化，去除装饰性渐变背景

**5.4 内容区扩展**：
- `.blog-article` / `.prose` 去除 max-width，让正文占满打印宽度

**5.5 链接 URL 显示**：

```css
/* 打印时在链接后显示 URL，便于纸质文档追溯 */
a[href^="http"]::after {
  content: ' (' attr(href) ')';
  font-size: 0.85em;
  color: #666;
  word-break: break-all;
}
```

**5.6 代码块打印优化**：保留浅灰底 + 边框 + 黑字，避免深色背景打印失败；行内代码去除主色，使用 `#000` + `#f0f0f0` 背景

**5.7 分页规则**：
- `h1-h4 { page-break-after: avoid }` - 标题不与正文分离
- `pre/blockquote/table/figure { page-break-inside: avoid }` - 块级元素不被截断

### 6. `src/pages/blog/[...slug].astro` - 博客详情页行内代码边框同步

**问题**：博客详情页 `.blog-article__content code` 原本无边框（仅 background + padding），与本次优化后的全局 `.prose code`（已添加边框）风格不一致。`.blog-article__content pre code` 也未显式重置边框。

**优化**：同步全局 prose 的修改：

```css
.blog-article__content code {
  /* 其余属性保持不变 */
  border: 1px solid var(--color-border);
  padding: 1px 6px; /* 原 2px 6px */
}

.blog-article__content pre code {
  /* 其余属性保持不变 */
  border: 0;
}
```

### 行内代码视觉风格

| 方面 | 优化前 | 优化后 |
| --- | --- | --- |
| prose 行内代码 | 仅背景色，无边框 | 背景 + 边框，与博客详情页一致 |

| 方面 | 优化前 | 优化后 |
| --- | --- | --- |

| 视觉一致性 | 与表单/按钮焦点反馈不一致 | 与全站焦点反馈风格统一 |
| 无障碍 | 键盘用户焦点辨识度低 | 焦点清晰可见，贴合折叠形态 |

### 范围滑块跨浏览器一致性

| 方面 | 优化前 | 优化后 |

### 打印体验

| 方面 | 优化前 | 优化后 |
| --- | --- | --- |
| 打印内容 | 包含 header/footer/按钮/CTA 等交互元素 | 仅保留正文与必要视觉元素 |
| 链接 URL | 丢失 | 打印在链接文本后 |
| 代码块 | 暗色背景打印失败/耗墨 | 浅灰底 + 黑字，打印友好 |
| 分页 | 元素可能被截断 | 标题与块级元素避免分页 |
| 容器宽度 | 受 max-width 限制 | 占满打印区域 |
| 暗色模式 | 影响打印 | 强制白底黑字 |
