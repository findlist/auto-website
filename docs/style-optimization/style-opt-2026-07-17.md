## 具体修改内容

### 1. `src/styles/global.css` - 按钮显式焦点环

**问题**：原 `.btn:focus-visible` 未定义，按钮在键盘 Tab 导航时依赖浏览器默认 outline，与表单元素（input/textarea/select）使用的 `var(--focus-ring)` box-shadow 焦点反馈不一致；且按钮的 hover 状态（阴影 + 边框变化）会掩盖默认 outline，导致键盘用户难以辨识当前焦点位置。

**优化**：在全局 `:focus-visible` 规则后新增 `.btn:focus-visible` 规则，统一使用 `var(--focus-ring)` box-shadow 作为焦点反馈：

```css
/* 按钮显式焦点环：与表单元素统一的 box-shadow 焦点反馈，避免 hover 状态掩盖键盘焦点 */
.btn:focus-visible {
  outline: none;
  box-shadow: var(--focus-ring);
}
```

### 2. `src/styles/global.css` - 触摸设备 hover 优化

**问题**：触摸设备（手机/平板）上点击元素后会触发 `:hover` 状态，由于没有鼠标"移出"事件，hover 样式会"粘住"直到用户点击其他位置，导致 `transform: translateY(-4px)` 上浮、`box-shadow` 放大等效果残留，影响后续视觉判断。

**优化**：新增 `@media (hover: none)` 媒体查询块，统一去除桌面专属的 transform 上浮与阴影放大效果，保留颜色/边框变化作为触摸反馈。覆盖以下元素：

- `.tool-card` / `.blog-card` / `.blog-section__card` / `.feature` - 卡片类上浮效果
- `.post-nav__link` - 文章导航链接上浮
- `.jsontool` / `.jwttool__seg` / `.hashtool__dropzone` / `.hashtool__result-item` - 工具组件上浮
- `.btn--primary` - 主按钮上浮
- `.colortool__palette-color` - 颜色卡片上浮

### 3. `src/styles/global.css` - 补充 `.prose h4` 样式

**问题**：原 `.prose` 样式已覆盖 h1/h2/h3，但缺少 h4 样式。在关于页、隐私政策页等长文章中，h4 直接回退为浏览器默认样式（粗体 + 1em），视觉权重与正文区分度不足，且与 h2（主色渐变细条）、h3（小圆点前缀）的装饰风格脱节。

**优化**：在 `.prose h3` 后新增 `.prose h4` 样式，使用主色小圆点前缀装饰，与 h3 形成层次差异，避免视觉过重：

```css
.prose h4 {
  font-size: 15.5px;
  margin: 20px 0 8px;
  font-weight: 600;
  color: var(--color-text-soft);
  position: relative;
  padding-left: 14px;
}

.prose h4::before {
  content: '';
  position: absolute;
  left: 0;
  top: 50%;
  transform: translateY(-50%);
  width: 6px;
  height: 6px;
  border-radius: 50%;
  background: var(--color-primary);
  opacity: 0.7;
}
```

### 4. `src/styles/global.css` - 博客列表/标签页 hero 装饰统一

**问题**：首页 hero 与工具页 hero 均有底部主色渐变细线装饰（`::after` 伪元素 + underline-expand 入场动画），但博客列表页 `[...page].astro`、博客标签索引页 `blog/tag/index.astro`、标签详情页 `blog/tag/[tag].astro` 使用的 `.page-hero`（无 `--tool` 修饰符）缺少该装饰，导致博客相关页面与全站其他 hero 视觉语言不一致。

**优化**：为 `.page-hero:not(.page-hero--tool)` 新增 `::after` 装饰，让博客列表/标签页的 hero 也具备底部主色渐变细线 + underline-expand 入场动画，与首页 hero、工具页 hero 视觉语言保持一致。

### 5. `src/pages/blog/[...slug].astro` - 补充 h4 样式

**问题**：博客详情页 `.blog-article__content` 已覆盖 h2/h3，但同样缺少 h4 样式。长文章中 h4 直接回退为浏览器默认样式，与全局 prose h4 风格脱节。

**优化**：在 `.blog-article__content h3` 后补充 `.blog-article__content h4` 与 `::before` 样式，与全局 prose h4 风格保持一致：

```css
/* h4 使用主色小圆点前缀，与 prose 全局风格一致，避免长文章中 h4 视觉过重 */
.blog-article__content h4 {
  font-size: 15.5px;
  margin: 22px 0 8px;
  font-weight: 600;
  color: var(--color-text-soft);
  position: relative;
  padding-left: 14px;
}

.blog-article__content h4::before {
  content: '';
  position: absolute;
  left: 0;
  top: 50%;
  transform: translateY(-50%);
  width: 6px;
  height: 6px;
  border-radius: 50%;
  background: var(--color-primary);
  opacity: 0.7;
}
```

### 6. `src/pages/about.astro` - article 子元素缩进修复

**问题**：`<article class="prose">` 内部的子元素（header、p、h2、ul 等）缩进不一致，原本为 4 空格，与外层 article 标签的 6 空格缩进不对齐，影响代码可读性。

**优化**：将 article 内部所有子元素统一缩进为 6 空格，与外层 article 标签对齐。此为纯代码风格修复，不影响渲染结果。
