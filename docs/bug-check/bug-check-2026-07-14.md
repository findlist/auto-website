## 记录的轻微问题（P2，未修复）

1. **AnimationTool.tsx:434-435** — 关键帧列表使用数组索引 `key={i}` 作为 React key。删除中间关键帧时，受控 input 的 DOM 节点按 key 复用会导致焦点/选区跳到错误的帧上。不影响数据正确性（controlled component 保证值正确），仅影响交互体验。建议修复方式：给 `KeyframeState` 增加唯一 `id` 字段，改用 `key={kf.id}`。

2. **jwtVerify.ts:310** — `formatRelativeTime` 函数中 `已${isFuture ? '' : '过期 '}` 存在冗余判断。该处位于外层三元表达式的 else 分支（`isFuture` 必为 false），内层 `isFuture ? '' : '过期 '` 恒为 `'过期 '`，结果正确但代码冗余。建议简化为 `已过期 ${days} 天 ${hours} 小时`。

3. **jwtVerify.ts:321 / formatRelativeTime** — `formatRelativeTime` 被 exp / nbf / iat 三处复用，对 iat（签发时间，过去时）使用"已过期 X 天"描述语义不准确（应为"X 天前"）。不影响验签结果与状态判定（iat 的 `status` 仍为 `valid`，`message` 仍为"签发时间正常"），仅 `relative` 字段文案不理想，且 `message` 字段同时显示正常提示，不会误导用户。

4. **clipboard.ts:39** — `document.execCommand('copy')` 已被 Web 标准废弃（astro check 提示 ts(6387)）。该调用为非安全上下文（HTTP）下的必要降级方案，在 `navigator.clipboard` 不可用时保证复制功能可用，属有意保留，非 bug。
