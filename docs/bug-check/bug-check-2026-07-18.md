### 修复详情

**问题文件**：`src/components/TrigonometricTool.tsx`（polar 预设的 CSS 生成代码）

**修改前**：
```tsx
/* atan2(y, x) 返回弧度，需用 calc 转换为度数后再旋转 */
transform: rotate(calc(atan2(var(--mouse-y), var(--mouse-x)) * 1rad));
```

**修改后**：
```tsx
/* atan2(y, x) 返回弧度，需用 calc 转换为度数后再旋转；
   指针默认指向下方（南），atan2 从正 X 轴（东）起算，因此需减 90deg 校正方向 */
transform: rotate(calc(atan2(var(--mouse-y), var(--mouse-x)) * 1rad - 90deg));
```

**根因分析**：
- atan2(y, x) 的角度起算方向为正 X 轴（即东方，0°）
- 而指针元素默认朝向为下方（即南方，对应 90°）
- 二者相差 90°，直接使用 atan2 的结果会导致指针方向偏离鼠标位置 90°
- 组件内预览 HTML 中已正确使用 `rotate(${v.angle - 90}deg)`，但生成的 CSS 代码（供用户复制使用）漏掉了 `-90deg` 偏移，造成「预览效果正确，但复制的 CSS 错误」的隐蔽 bug
